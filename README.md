# ProxyPulse

**Живые MTProto-прокси для Telegram · только Европа и Америка**

ProxyPulse — это нативное Android-приложение (Kotlin + Jetpack Compose), которое
собирает MTProto-прокси из открытых источников, **проверяет каждую на
работоспособность в реальном времени**, оставляет только живые и только из
разрешённых регионов (Европа и Америка), и обновляет весь список раз в час —
нерабочие прокси удаляются автоматически при каждом обновлении.

---

## Что внутри

- **Парсинг** нескольких публичных источников MTProto-прокси (host, port, secret).
- **Фильтр по региону**: остаются только прокси из стран Европы и Америки.
  Прокси без определённой страны отбрасываются.
- **Проверка «живости»**: для каждой прокси делается реальное TCP-подключение
  (таймаут 4 с, до 40 параллельных проверок). Замеряется пинг. В список попадают
  **только те, что ответили** — это и есть «сразу удаляются нерабочие».
- **Сортировка по пингу** — самые быстрые сверху.
- **Кнопка «Добавить в Telegram»**: открывает Telegram по ссылке
  `tg://proxy?server=…&port=…&secret=…`, и Telegram сам предлагает включить прокси.
  Если приложение Telegram не установлено — открывается веб-ссылка `https://t.me/proxy?...`.
- **Кнопка копирования** ссылки прокси в буфер обмена.
- **Автообновление раз в час** через WorkManager (работает в фоне), плюс
  обновление при запуске, если данные устарели, плюс ручной pull-to-refresh.
- **Тёмный дизайн** с фирменным градиентом и анимированным логотипом-пульсом.

## Название и логотип

**ProxyPulse** — «пульс прокси». Метафора простая: приложение постоянно
«щупает пульс» каждой прокси и показывает только живые. Отсюда и логотип —
линия кардиограммы (ЭКГ) с бегущей светящейся точкой, переходящая в бумажный
самолётик Telegram. Фирменные цвета: бирюзовый `#22D3EE` → индиго `#6366F1`
на тёмно-синем фоне `#0B0E1A`.

Логотип живой: в приложении точка пульса бежит по линии (см. `ui/components/Logo.kt`),
а иконка приложения — адаптивная (`res/mipmap-anydpi-v26/`).
Превью логотипа — файл `logo_preview.png` рядом с архивом.

## Как собрать и запустить

> ⚠️ Готовый APK здесь **не приложен** — его нельзя скомпилировать без Android SDK.
> Проект рассчитан на сборку в Android Studio в один клик.

1. Установите **Android Studio** (Ladybug 2024.2 или новее).
2. **File → Open** → выберите распакованную папку `ProxyPulse`.
3. Android Studio сама подтянет Gradle Wrapper, Android SDK и зависимости
   (нужен интернет при первой сборке — Google Maven и Maven Central).
4. Нажмите **Run** (▶) — приложение соберётся и установится на устройство/эмулятор.
   Минимальная версия Android — 7.0 (API 24).

### Про Gradle Wrapper

В архиве есть `gradle/wrapper/gradle-wrapper.properties`, но **нет бинарника
`gradle-wrapper.jar`** (его нельзя сгенерировать в среде сборки без Gradle).
Это не проблема:

- Android Studio при первом открытии проекта создаёт `gradle-wrapper.jar`
  автоматически.
- Либо, если у вас локально установлен Gradle, выполните в папке проекта:
  ```
  gradle wrapper --gradle-version 8.11.1
  ```
  после чего станут доступны `./gradlew assembleDebug` и т.д.

## Где что настраивается

| Что | Файл |
|---|---|
| Источники прокси (список URL) | `app/src/main/java/com/proxypulse/app/data/remote/ProxySource.kt` → `SOURCES` |
| Список разрешённых стран (Европа/Америка) | `app/src/main/java/com/proxypulse/app/data/region/Regions.kt` |
| Период автообновления (по умолчанию 1 час) | `data/repository/ProxyRepository.kt` → `MAX_AGE_MS` и `core/ProxyPulseApp.kt` (PeriodicWork) |
| Таймаут/параллелизм проверки | `data/check/ProxyChecker.kt` |
| Цвета и тема | `app/src/main/java/com/proxypulse/app/ui/theme/` |

**Добавить свой источник** прокси: впишите URL в список `SOURCES`. Парсер
понимает JSON-массивы/объекты, строки `tg://`-ссылок и формат `host:port:secret`.

## Ответственное использование

Приложение лишь собирает и проверяет публично опубликованные прокси-серверы.
Работоспособность и доверие к конкретной прокси — на усмотрение пользователя:
MTProto-прокси видит ваш трафик на уровне маршрутизации, поэтому подключайтесь
только к тем, которым доверяете. Используйте в соответствии с законами вашей
страны и правилами Telegram.
